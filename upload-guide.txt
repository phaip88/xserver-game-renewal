# GitHub 上传指南

## 需要上传的文件列表：

必须上传：
1. minecraft.java              - 主程序源代码
2. META-INF/MANIFEST.MF        - JAR 清单文件
3. .github/workflows/build.yml - GitHub Actions 构建配置

推荐上传：
4. .gitignore                  - Git 忽略配置
5. README.md                   - 项目说明文档
6. build.bat                   - Windows 构建脚本（可选）

## Git 命令行上传步骤：

# 1. 初始化 Git 仓库
git init

# 2. 添加文件
git add minecraft.java
git add META-INF/MANIFEST.MF
git add .github/workflows/build.yml
git add .gitignore
git add README.md
git add build.bat

# 3. 提交
git commit -m "Initial commit: Xray-Sing proxy server with auto-build"

# 4. 关联远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/你的仓库名.git

# 5. 推送到 GitHub
git push -u origin main

# 如果提示分支名称错误，使用：
git branch -M main
git push -u origin main

## 验证上传成功：

1. 访问你的 GitHub 仓库页面
2. 确认所有文件都已上传
3. 点击 "Actions" 标签查看自动构建状态
4. 等待构建完成（约 1-2 分钟）
5. 从 "Actions" 或 "Releases" 下载 server.jar

## 常见问题：

Q: 提示 "remote: Permission denied"
A: 需要配置 GitHub 认证，使用 Personal Access Token 或 SSH Key

Q: 构建失败
A: 检查 Actions 日志，确认 minecraft.java 文件已正确上传

Q: 找不到 server.jar
A: 进入 Actions → 选择最新构建 → 滚动到底部下载 Artifacts
